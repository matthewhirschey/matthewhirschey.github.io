# matthewhirschey.com — project guide

Personal site for Matthew Hirschey, deployed via GitHub Pages (custom domain `matthewhirschey.com`, CNAME already in place). Karpathy.ai-inspired single-page layout.

## Hard constraints

- **Zero frameworks.** Pure HTML + CSS. No JS. No bootstrap. No analytics.
- **Output stays tiny.** `index.html` should be ~17 KB, `style.css` ~5 KB. If either grows past 30 KB, something is wrong.
- **No 500-pound websites.** If a change pulls in a dependency, find another way.

## Source → build → output

```
index.qmd   ──┐
template.html ─┤──[ quarto render ]──►  index.html  ──►  GitHub Pages
render-data.lua ┘
```

Edit `index.qmd` (and for some things, `template.html` or `render-data.lua`). Run `make build`. Commit both the source and the regenerated `index.html`.

```
make build   # quarto render + prepend doctype
make open    # build then open in browser
make clean   # delete index.html
```

## Files

| File | Role |
|---|---|
| `index.qmd` | **Source of truth.** YAML front-matter (hero, links, talks, projects, writing, publications) + markdown body |
| `template.html` | Pandoc template — hero scaffold, `$body$` slot, footer |
| `render-data.lua` | Pandoc Lua filter that expands `{{talks}}`, `{{projects}}`, `{{writing}}`, and `{{publications}}` markers into HTML from YAML arrays. `{{publications}}` also emits one `<script type="application/ld+json">` ScholarlyArticle block per entry with a `doi:` (HTML output only — gated on `FORMAT == "html"` so the GFM mirror stays clean). |
| `style.css` | Pure CSS, karpathy-derived 12-col grid, `.entry` timeline, `.card`, `.pub` |
| `index.html` | **Build artifact.** Committed so GitHub Pages can serve it. Never edit by hand. |
| `index.md` | **Build artifact.** Plain-markdown mirror of the homepage, generated by Quarto's `gfm` format for LLM crawlers. Referenced from `llms.txt`. |
| `llms.txt` | Static [llms.txt](https://llmstxt.org) index at site root — H1, blockquote summary, links. Hand-maintained. |
| `sitemap.xml` | Static, four URLs. Referenced from `robots.txt`. |
| `robots.txt` | Permissive, welcomes major LLM bots (GPTBot, ClaudeBot, PerplexityBot, Google-Extended, CCBot, Applebot-Extended). |
| `Makefile` | `make build` / `open` / `clean` |
| `img/` | Timeline logos + any future section images |
| `Hirschey_CV.pdf` | Copied from `~/everything/work/areas/resume-cv` on each CV update |
| `CNAME`, `404.html`, `refresh` | GitHub Pages plumbing, don't touch |

## Editing patterns

- **Hero tagline, links, email:** YAML front-matter at top of `index.qmd`.
- **Talks:** add one YAML entry under `talks:` — fields `year`, `venue`, `loc`.
- **Projects / tools:** add one YAML entry under `projects:` — fields `title`, `url`, `desc` (markdown OK), plus either `logo:` (path to an image in `img/`) or `badge:` (short text fallback).
- **Writing cards:** add one YAML entry under `writing:` — fields `title`, `url`, `img` (thumbnail path), `desc`. Rendered as a 3-up card row from a `{{writing}}` marker in the body.
- **Publications:** add one YAML entry under `publications:` — fields `title`, `authors` (string; escape `*` co-first markers as `\\*`), `venue`, `year`, `doi` (publisher DOI URL — omit for non-DOI'd entries; ScholarlyArticle JSON-LD only emits when `doi` is set). Append new ones at the top of the array. The visible title becomes a clickable DOI link when `doi` is present.
- **Timeline:** `:::::: {.entry .row}` blocks — note **6 colons** on the outer fence, **3** on the inner. Copy an existing block and edit year / logo path / description.
- **Prose sections (bio, teaching, writing intro, misc):** plain markdown paragraphs and lists.

After any edit: `make build`, eyeball `index.html` in a browser, commit both files, push.

## LLM SEO — what's wired up

The site is optimized for AI search and LLM crawlers. Do not break these unless you know what you're doing:

- **`<head>` metadata** — Open Graph, Twitter Card, canonical URL, author meta. All driven by YAML fields in `index.qmd` (`canonical_url`, `og_image`, `twitter_handle`, `description`).
- **JSON-LD Person schema** — inlined in `template.html` as a `<script type="application/ld+json">` block. Pulls `sameAs` from the `same_as:` YAML array. If you add a new identity link, put it in `same_as:` and rebuild.
- **`llms.txt`** — hand-written at root per the [spec](https://llmstxt.org). Minimal form: H1, blockquote, `## About` section.
- **`index.md`** — auto-generated by Quarto's dual `html` + `gfm` format. LLM-preferred plain markdown mirror. Zero maintenance.
- **`sitemap.xml` + `robots.txt`** — static. Edit only when adding/removing top-level pages.

**Note on `&` in YAML:** the Lua filter's `json_escape()` now emits `&`, `<`, `>` Unicode escapes for `&`, `<`, `>` so JSON-LD survives Pandoc's HTML escaping and the browser's `</script>` scan. You can use `&` in titles and venues without breaking JSON parse — but the visible HTML will still render `&amp;` (correct HTML escaping), so prefer "and" spelled out when it reads naturally.
